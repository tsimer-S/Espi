What it does

EVERY FEATURE EXPLAINED

FitBot combines an AI nutrition assistant with a structured strength program in one Telegram chat window.

🍽
Calorie & Macro Tracking

Type what you ate in plain language — no templates, no barcodes. The AI understands "bowl of oatmeal with banana" and instantly returns calories, protein, fat, and carbs with a personalized tip.

LLM-powered
📊
Daily Food Diary

Every meal you log is stored in your personal diary. At any point you can view the full day's log with a running calorie total. Use /clear to reset it each morning.

session storage
🔥
Personalized Calorie Norm

Based on your height, weight, age, and gender, FitBot calculates your Basal Metabolic Rate (Mifflin–St Jeor formula) and shows calorie targets for four activity levels — from sedentary to very active.

BMR formula
💪
3-Week Bench Press Program

Enter your one-rep max and FitBot calculates every weight for every session across three weeks: Week 1 builds volume, Week 2 adds intensity, Week 3 peaks with a max attempt day.

% of 1RM
📅
Session-by-Session Navigation

The bot tracks exactly which week and day you're on. Hit "Next" after each workout to advance the program automatically — Tuesday strength day, Saturday volume day, and so on.

state tracking
👤
User Profile & BMI

On first launch, FitBot collects your height, weight, age, gender, and bench max through a guided conversation. It stores this to personalize every calorie estimate and training recommendation.

onboarding flow

Bilingual — RU & EN

Switch between Russian and English with a single button tap. All menus, workout plans, calorie outputs, and AI responses adapt instantly to the selected language.

i18n
🤖
FitBot Persona

The bot has a defined identity — it knows its name, its purpose, and its boundaries. Ask it about politics or movies and it politely redirects you back to fitness. Ask who made it and it answers honestly.

Database

Storage Approach

FitBot uses in-memory Python dictionaries (no external database) keyed by Telegram user ID. Each user's profile, diary, current week, and language preference is stored in a dict that lives for the session. This keeps the MVP simple and stateless — a future version could persist data to SQLite or Redis.

Model Selection

Why LLaMA 3.3 70B on Groq

We chose LLaMA 3.3 70B Versatile via Groq for three reasons:

1. Free tier. Groq provides generous free API access, making the project cost-zero for development and demonstration.


2. Speed. Groq's custom LPU hardware delivers very low latency — responses arrive in under a second, which feels natural for a Telegram chat interface.


3. Instruction following. The 70B model reliably adheres to strict output formats (the calorie breakdown template) without hallucinating extra text, which was critical for parsing the response back into the diary.

Prompt Engineering

Key Design Choices

Three deliberate decisions shaped how the bot responds:

1. Strict format enforcement. The prompt instructs the model to reply only in the calorie breakdown template. This made it possible to parse the calorie number from the text and store it in the diary automatically.
2. Dynamic language injection. Instead of asking the model to detect language freely (unreliable), we inject the target language directly into the system prompt — Reply in English / Reply in Russian — based on the user's stored preference. This eliminated mixed-language responses.
3. Profile context injection. Every calorie request includes the user's height, weight, age, gender, and bench max in the prompt. This allows the model to give a personalized tip rather than a generic one — for example, suggesting higher protein intake to someone with a heavy 1RM.
4. Identity & scope guardrails. The system prompt establishes a clear persona (FitBot) and limits the model to fitness topics only. Off-topic queries are intercepted before reaching the API where possible; the model handles edge cases.
